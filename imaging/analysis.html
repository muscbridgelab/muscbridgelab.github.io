Image analysis starts with MRI data acquisiton. Once data is acquired, it needs to be downloaded from the scanner server on which it is temporarily stored. From there, data should be checked for any quality control issues. Finally, dicoms should be sorted, converted to nifti, and then data should be preprocessed.

Required Software: Filezilla (or another FTP software), Horos, dicomSort, MRIcoGL, Matlab, PyDesigner, and Freesurfer

  1. Download raw data from CBIHome
    1. Access CBIHome through an FTP client (requires MUSC wifi or VPN)
      • sftp://cbihome.musc.edu
      • Log in with NETID and PW
    2. Navigate to MRdata folder, then to the folder of the PI of the study, then to the study folder, then click on the ‘upload’ folder.
    3. You will see a series of .zip folders, download whichever you need.

  2. Raw data QCing - Always, ALWAYS check your raw data for quality control issues.
    1. Unzip the .zip dicom folder
    2. Open Horos and drag the entire unzipped dicom folder into the program
    3. Select ‘copy the links’
    4. Scroll through scans and note QC issues as applicable

  3. Sort and convert raw dicoms
    1. Sort dicoms
    2. Convert raw data from dicom to nifty using dcm2niix
